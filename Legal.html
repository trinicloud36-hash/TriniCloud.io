Privacy Policy

üîí Your Privacy is Our Priority

This Privacy Policy explains how Jogania collects, uses, and protects your information when you use our enterprise AI platform.

1. Information We Collect

1.1 Account Information

When you create a Jogania account, we collect:

  • Business Information: Company name, industry, business size, contact details
  • User Information: Name, email address, phone number, job title
  • Authentication Data: Password (encrypted), two-factor authentication settings
  • Billing Information: Payment methods, billing addresses, transaction history

1.2 Usage Data

We automatically collect information about how you use our platform:

  • Platform Analytics: Feature usage, session duration, click patterns, error reports
  • AI Agent Data: Training data sources, conversation logs, performance metrics
  • Technical Information: IP addresses, browser types, device information, API usage
  • Integration Data: WhatsApp Business connections, website widget deployments

1.3 Training Data

Your AI agents are trained on data you provide:

  • Website Content: Text, images, documents from URLs you submit for training
  • Custom Knowledge: Files, documents, and manual content you upload
  • Conversation Data: Chat logs between your agents and customers (for improvement)

2. How We Use Your Information

2.1 Service Provision

  • Creating and managing your AI agents
  • Processing and responding to customer inquiries via your agents
  • Providing analytics and performance insights
  • Facilitating integrations with WhatsApp, websites, and APIs

2.2 Platform Improvement

  • Analyzing usage patterns to enhance features
  • Training our underlying AI models (using anonymized, aggregated data only)
  • Identifying and fixing bugs or technical issues
  • Developing new features and capabilities

2.3 Communication

  • Sending service updates, security alerts, and technical notifications
  • Providing customer support and technical assistance
  • Sharing product updates and new feature announcements
  • Processing billing and subscription communications

3. Data Sharing and Disclosure

‚úÖ We Never Sell Your Data

Jogania does not sell, rent, or lease your personal information or business data to third parties for marketing purposes.

3.1 Service Providers

We share data with trusted third-party providers who help us operate our platform:

  • Cloud Infrastructure: AWS, Google Cloud (for secure hosting and storage)
  • Payment Processing: Stripe, PayPal (for subscription billing)
  • Communication Services: SendGrid (for email notifications)
  • Analytics: Mixpanel (for anonymized usage analytics)

3.2 Legal Requirements

We may disclose information when required by law or to:

  • Comply with legal processes, subpoenas, or court orders
  • Protect the rights, property, or safety of Jogania, users, or the public
  • Investigate potential violations of our Terms of Service
  • Prevent fraud, security breaches, or other illegal activities

4. Data Security

4.1 Encryption

  • In Transit: All data transmitted using TLS 1.3 encryption
  • At Rest: Database and file storage encrypted using AES-256
  • API Communications: HTTPS only with certificate pinning

4.2 Access Controls

  • Multi-Factor Authentication: Required for all admin accounts
  • Role-Based Access: Least privilege principle for all team members
  • Audit Logging: Complete logs of all data access and modifications

4.3 Infrastructure Security

  • SOC 2 Type II: Annually audited security controls
  • Vulnerability Scanning: Regular automated security assessments
  • Incident Response: 24/7 monitoring and response procedures

5. Your Rights and Choices

5.1 Data Access

You have the right to:

  • Access all personal data we hold about you
  • Download your agent training data and conversation logs
  • Receive a copy of your data in a portable format

5.2 Data Correction

  • Update your account information at any time through your dashboard
  • Correct inaccurate personal or business information
  • Modify AI agent training data and knowledge bases

5.3 Data Deletion

  • Delete your account and all associated data
  • Remove specific training data or conversation logs
  • Request complete data erasure (subject to legal retention requirements)

6. International Data Transfers

Jogania operates globally and may transfer data between countries. We ensure all transfers comply with applicable data protection laws through:

  • Adequacy Decisions: Transfers to countries with adequate protection
  • Standard Contractual Clauses: EU-approved data transfer agreements

  • Auto-Scaling: Dynamic resource allocation based on demand
  • Load Balancing: Distributed traffic management for high availability
  • Container Security: Kubernetes with security policies and network isolation

1.2 Network Security

  • Virtual Private Cloud (VPC): Isolated network environments
  • Web Application Firewall (WAF): Protection against common attacks
  • DDoS Protection: AWS Shield Advanced for distributed attack mitigation
  • Intrusion Detection: Real-time monitoring and alerting

2. Data Protection

2.1 Encryption Standards

  • Data at Rest: AES-256 encryption for all stored data
  • Data in Transit: TLS 1.3 for all communications
  • Database Encryption: Transparent data encryption (TDE)
  • Key Management: AWS Key Management Service (KMS) with rotation

2.2 Data Classification

  • Public: Marketing materials, public documentation
  • Internal: Business operations data, analytics
  • Confidential: Customer data, training materials
  • Restricted: Authentication credentials, encryption keys

3. Access Management

3.1 Authentication

  • Multi-Factor Authentication (MFA): Required for all admin accounts
  • Single Sign-On (SSO): SAML 2.0 and OAuth 2.0 support
  • Password Policy: Minimum complexity requirements and rotation
  • Session Management: Secure session tokens with timeout

3.2 Authorization

  • Role-Based Access Control (RBAC): Granular permission system
  • Principle of Least Privilege: Minimum necessary access rights
  • Just-in-Time Access: Temporary elevated privileges for maintenance
  • Regular Access Reviews: Quarterly audit of user permissions

4. Monitoring and Logging

4.1 Security Monitoring

  • 24/7 SOC: Security operations center monitoring
  • SIEM Integration: Security information and event management
  • Threat Intelligence: Real-time threat detection and response
  • Anomaly Detection: AI-powered behavioral analysis

4.2 Audit Logging

  • Comprehensive Logs: All user actions and system events
  • Tamper-Proof Storage: Immutable log storage for compliance
  • Log Retention: 7 years for security logs, 3 years for operational logs
  • Real-Time Alerting: Immediate notification of security events

5. Incident Response

5.1 Response Team

  • Security Team: Dedicated cybersecurity professionals
  • DevOps Team: Infrastructure and application specialists
  • Legal Team: Compliance and regulatory expertise
  • Communications Team: Customer and stakeholder notification

5.2 Response Procedures

  • Detection and Analysis: Initial threat assessment within 1 hour
  • Containment: Immediate isolation of affected systems
  • Eradication: Removal of threats and vulnerabilities
  • Recovery: System restoration and service resumption
  • Lessons Learned: Post-incident review and improvements

6. Compliance and Certifications

6.1 Security Frameworks

  • SOC 2 Type II: Annual third-party security audit
  • ISO 27001: Information security management system
  • NIST Cybersecurity Framework: Risk-based security controls
  • OWASP Top 10: Web application security best practices

6.2 Regular Assessments

  • Penetration Testing: Quarterly external security testing
  • Vulnerability Scanning: Daily automated security scans
  • Code Security Review: Static analysis for all code changes
